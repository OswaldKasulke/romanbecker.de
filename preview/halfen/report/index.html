5. Tracking-Probleme

Befund 5.1 — Google-Ads-Conversion-Tracking unzuverlässig

⚠ KRITISCH

Eine PMax-Kampagne (PMAX HALFEN DE 05-2026) meldet 590 Conversions auf nur 60 Klicks — das wären knapp 10 Käufe pro Klick. Physikalisch unmöglich.

Mögliche Ursache (zu verifizieren): Mehrfach-Tracking durch parallele Setups. Im settings_data.json sind aktiv: Shopify Google Channel App, Simprosys, plus möglicherweise Google-Ads-eigener Tag im Theme. Welche Tags konkret laufen, muss im Shopify Admin (Custom-Metafields foxtheme.code_head/code_body) verifiziert werden.

Server-Side-Tracking-Empfehlung (3 Stufen)

→ STUFE 1 — QUICK-WIN (heute, kostenlos)

Shopify Conversions API für Meta aktivieren. Toggle in Shopify-Admin → Marketing → Pixel Connection → Conversion API. Adressiert Express-Pay-Tracking-Lücke. Kosten: 0 €. Aufwand: 30-60 Min. Konkrete Halfen-Wirkung erst nach Aktivierung messbar (typische Industry-Schätzungen variieren stark, daher hier nicht beziffert).

→ STUFE 2 — MITTELFRISTIG (Wo 2-4)

GA4 Measurement Protocol via Cloud Function + Shopify Web Pixel API. Eigene Cloud-Function (Free-Tier), die GA4-Events serverseitig sendet. Kosten: ~0 € laufend. Aufwand: 8 h Setup. Konkrete Halfen-Wirkung erst nach Implementierung messbar.

→ STUFE 3 — OPTIONAL (Wo 5+)

Stape.io oder Server-GTM für volle Kontrolle. Kosten: 50-200 €/Monat. Aufwand: 20+ h. Nur wenn Stufe 1+2 nicht reichen.
